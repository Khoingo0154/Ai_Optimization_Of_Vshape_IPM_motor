# Ansys Maxwell Batch Simulation Script (MATLAB)



## Initial NOTES ##

This Script has been created with the assumption that the Python script is going to generate a set of valid design variables (within the design variable limits), ping the MATLAB script to run the simulations in Ansys and then read the output data generated by each simulation to decide on the next generation of design variable sets. If this assumption is wrong I can make the required changes after a clarification of the interaction between the Python script and the MATLAB script.

Read in full screen for some graphics to be viewed as expected.



## Overview ##

This MATLAB script automates batch simulation of a Maxwell (Ansys Electronics Desktop) motor design, via ActiveX/COM. 
This lets you sweep many parameter combinations without manually editing the design in Maxwell each time.



## Input Files ##

Both input files must be located in the same folder as the script.


File 1: 
Name: `Ai_Optimization_Bounds.xlsx`

Defines the 19 design variables and their limits. Expected columns:

+---------------+-----------------------------------------------------------------------+
|    Column     | 				Description 				|
+---------------+-----------------------------------------------------------------------+
|  `Parameter` 	|    		Variable name as defined in the Maxwell design     	|
| `Lower_Limit` | 		 	  Minimum allowed value 			|
| `Upper_Limit` | 			  Maximum allowed value 			|
|    `Step` 	| 			Step size for the variable 			|
|    `Unit` 	|    Unit string (e.g. `mm`, `deg`); leave blank for unitless values 	|
+---------------+-----------------------------------------------------------------------+


This file is currently used to pull variable names and units — the lower/upper/step columns are read in but not yet applied as constraints in the script. This was done as an assumption that the Python script would generate the variables based on the Upper and Lower Limits provided in the file and the validation check wouldn't be done in the MATLAB script (Can be changed if needed)


File 2:
Name: `Ai_Optimization_ParamValues.xlsx`

Contains the actual parameter sets to simulate. Each row is one simulation iteration, and each column corresponds to one of the 19 parameters (in the same order as `Ai_Optimization_Bounds.xlsx`). The number of rows can vary — one simulation is run per row.

## Workflow ##

1. %% Setup - Loads the bounds file (variable names/units) and the parameter values file.
2. %% Connect to Maxwell - Starts (or attaches to) the `Ansoft.ElectronicsDesktop` ActiveX server, opens/activates the project (`Matlab_Ai_Optimization.aedt`), and sets the active design to `Vshape_IPM`.
3. %% Iterate - For each row in the parameter values table:
   	- Assigns each of the 19 variables via `SetVariableValue` (appending the unit string when one is provided).
   	- Resets the setup to time zero (`ResetSetupToTimeZero`).
   	- Runs the analysis (`Analyze`).
   	- Exports the output variables table to `output_vars_iter_<N>.csv` and reads it back into MATLAB as `outputData`.
4. %% Cleanup - Releases the ActiveX COM objects. The project is *not* saved or closed by default (see commented-out lines if you want that behavior).



## Outputs

## Customization Notes

- Saving/closing the project: commented out at the end of the script; uncomment `invoke(oProject, 'Save')` and/or `oDesktop.CloseProject(project_name)` if desired.
- Variable order: the order of columns in `Ai_Optimization_ParamValues.xlsx` must match the order of rows (parameters) in `Ai_Optimization_Bounds.xlsx`, since the script maps them positionally via the loop index `i`.
